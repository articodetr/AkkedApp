### 3. ✅ استخدام `.update()` بدلاً من `.upsert()`

**المشكلة:**
- الكود في `app/whatsapp-templates.tsx` كان يستخدم `.update()`
- إذا لم يكن السجل موجوداً، يفشل التحديث
- **النتيجة:** فشل حفظ قوالب الواتساب

**الإصلاح:**
- تم تغيير `.update()` إلى `.upsert()` مع `onConflict: 'id'`
- الآن يتم إنشاء السجل تلقائياً إذا لم يكن موجوداً

**الملفات المعدلة:**
- `/app/whatsapp-templates.tsx` - السطور 57-63

---

### 4. ✅ حساب الأرباح والخسائر يظهر في قوائم العملاء

**المشكلة:**
- حساب "الأرباح والخسائر" (`PROFIT_LOSS_ACCOUNT`) كان يظهر في قوائم اختيار العملاء
- هذا الحساب نظامي ولا يجب اختياره عند إضافة حوالات
- **النتيجة:** قد يختار المستخدم الحساب الخطأ عن طريق الخطأ

**الإصلاح:**
- تم إضافة filter `.neq('phone', 'PROFIT_LOSS_ACCOUNT')` في جميع الاستعلامات
- الملفات المعدلة:
  - `/app/new-movement.tsx` - السطر 131
  - `/app/new-transaction.tsx` - السطر 64

---

### 5. ✅ عدم وجود validation للأرقام السالبة

**المشكلة:**
- لا يوجد تحقق من أن المبلغ أو العمولة ليست سالبة
- يمكن للمستخدم إدخال أرقام سالبة تسبب مشاكل في الحسابات
- **النتيجة:** بيانات غير صحيحة في قاعدة البيانات

**الإصلاح:**
تم إضافة validations في:

**في `app/new-movement.tsx`:**
```typescript
if (parseFloat(formData.amount) < 0) {
  Alert.alert('خطأ', 'المبلغ لا يمكن أن يكون سالباً');
  return;
}

if (formData.commission && parseFloat(formData.commission) < 0) {
  Alert.alert('خطأ', 'العمولة لا يمكن أن تكون سالبة');
  return;
}

if (formData.commission && parseFloat(formData.commission) >= parseFloat(formData.amount)) {
  Alert.alert('خطأ', 'العمولة لا يمكن أن تكون أكبر من أو تساوي المبلغ');
  return;
}
```

**في `components/QuickAddMovementSheet.tsx`:**
- تم إضافة نفس validations

**الملفات المعدلة:**
- `/app/new-movement.tsx` - السطور 148-161
- `/components/QuickAddMovementSheet.tsx` - السطور 153-168
