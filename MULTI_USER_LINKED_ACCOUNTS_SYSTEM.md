# نظام ربط المستخدمين بالعملاء - Multi-User Linked Accounts System

## نظرة عامة

تم تطوير نظام متكامل يسمح للمستخدمين بربط بعضهم البعض كعملاء، مما يوفر إدارة شاملة للحسابات المتبادلة بين المستخدمين في التطبيق.

---

## الميزات الرئيسية

### 1. نظام المستخدمين المتعددين

- **كل مستخدم لديه قائمة عملاء خاصة به**
- **المدير يرى جميع البيانات**
- **المستخدمون يرون عملائهم فقط**
- **نظام أمان محكم مع Row Level Security (RLS)**

### 2. ربط المستخدمين كعملاء

#### إضافة عميل جديد - خياران:
1. **عميل عادي**: عميل غير مسجل في النظام (طريقة تقليدية)
2. **ربط مستخدم موجود**: ربط مستخدم مسجل كعميل

#### البحث عن المستخدمين:
- بحث برقم الحساب
- عرض النتائج مع حالة الربط (مربوط أو متاح)
- منع ربط نفس المستخدم مرتين
- منع ربط المستخدم لنفسه

### 3. الحسابات المتبادلة

#### صفحة خاصة للحسابات المتبادلة:
- عرض جميع المستخدمين المربوطين
- تمييز الدور: "عميلك" أو "أنت عميل عنده"
- عرض الأرصدة المتبادلة بوضوح
- الوصول السريع لصفحة تفاصيل العميل

#### منطق الأرصدة:
**إذا كنت المالك (العميل عندك):**
- رصيد موجب = له عندك
- رصيد سالب = لك عنده

**إذا كنت العميل (عند المستخدم الآخر):**
- رصيد موجب = لك عنده
- رصيد سالب = له عندك

### 4. تمييز العملاء المرتبطين

#### في قائمة العملاء:
- علامة "مستخدم" بجانب الاسم
- أيقونة Link على الصورة الرمزية
- خط أزرق على الجانب الأيسر للبطاقة

#### في صفحة تفاصيل العميل:
- Badge "مستخدم مسجل" في الهيدر
- معلومات إضافية عن المستخدم المرتبط

## التعديلات التقنية

### 1. قاعدة البيانات (Database)

#### تعديلات جدول `customers`:
```sql
- user_id uuid NOT NULL -- المستخدم المالك للعميل
- linked_user_id uuid NULL -- معرف المستخدم المرتبط (إذا كان مستخدم مسجل)
```

#### جدول جديد `user_customer_links`:
```sql
- id uuid PRIMARY KEY
- owner_user_id uuid NOT NULL -- المستخدم المالك
- linked_user_id uuid NOT NULL -- المستخدم المرتبط
- customer_id uuid NOT NULL -- معرف سجل العميل
- status text -- حالة الربط (active, inactive, blocked)
- created_at timestamptz
```

#### Views جديدة:
- `user_mutual_balances`: أرصدة متبادلة مفصلة بين المستخدمين
- `user_linked_accounts`: قائمة المستخدمين المرتبطين مع معلوماتهم

#### دوال جديدة:
```sql
-- البحث عن مستخدمين برقم الحساب
search_users_by_account_number(p_account_number, p_current_user_id)

-- إنشاء عميل مرتبط بمستخدم
create_linked_customer(p_owner_user_id, p_linked_user_id, p_customer_name)

-- تعيين المستخدم الحالي في السياق (لـ RLS)
set_current_user(user_name)
```

### 2. Row Level Security (RLS)

#### سياسات الأمان على `customers`:
- **القراءة**: المستخدم يرى عملاءه + نفسه كعميل عند الآخرين + المدير يرى الجميع
- **الإضافة**: المستخدم يضيف عملاء لنفسه فقط
- **التعديل**: المستخدم يعدل عملاءه فقط + المدير
- **الحذف**: المستخدم يحذف عملاءه فقط (مع حماية حساب الأرباح)

#### سياسات الأمان على `account_movements`:
- **القراءة**: حركات عملاء المستخدم + حركاته كعميل عند الآخرين
- **الإضافة/التعديل/الحذف**: حركات عملاء المستخدم فقط

### 3. الكود (Frontend)

#### ملفات محدثة:
1. **`types/database.ts`**: إضافة أنواع جديدة
   - `UserInfo`, `UserCustomerLink`, `UserMutualBalance`
   - `UserLinkedAccount`, `SearchUserResult`

2. **`contexts/AuthContext.tsx`**: إعداد current_user في Supabase
   - استدعاء `set_current_user` عند تسجيل الدخول
   - استدعاء `set_current_user` عند بدء التطبيق

3. **`app/add-customer.tsx`**: تحديث واجهة إضافة العميل
   - خيار اختيار النوع: عميل عادي أو ربط مستخدم
   - حقل بحث عن المستخدمين
   - عرض نتائج البحث
   - عرض المستخدم المحدد

4. **`app/(tabs)/customers.tsx`**: تمييز العملاء المرتبطين
   - أيقونة Link على الصورة الرمزية
   - Badge "مستخدم"
   - خط أزرق على البطاقة

5. **`app/customer-details.tsx`**: عرض معلومات المستخدم المرتبط
   - Badge "مستخدم مسجل" في الهيدر

6. **`app/(tabs)/settings.tsx`**: إضافة رابط للحسابات المتبادلة

#### ملفات جديدة:
1. **`app/linked-accounts.tsx`**: صفحة الحسابات المتبادلة
   - عرض جميع المستخدمين المرتبطين
   - تمييز الدور والأرصدة
   - وصول سريع لتفاصيل العميل

## سيناريوهات الاستخدام

### سيناريو 1: جلال يضيف أحمد كعميل

1. **جلال** يذهب لصفحة "إضافة عميل"
2. يختار "ربط مستخدم موجود"
3. يبحث برقم حساب أحمد (مثلاً: 26001)
4. يظهر أحمد في النتائج
5. جلال يختار أحمد ويحفظ
6. **النتيجة**:
   - أحمد يظهر في قائمة عملاء جلال مع علامة "مستخدم"
   - أحمد يمكنه رؤية حسابه عند جلال من صفحة "الحسابات المتبادلة"

### سيناريو 2: جلال يضيف حركة لأحمد

1. **جلال** يدخل لصفحة تفاصيل أحمد (عميله)
2. يضيف حركة "صادرة" بمبلغ 500 دولار (جلال أعطى أحمد)
3. **النتيجة عند جلال**:
   - يظهر "له عليك 500 دولار" (أحمد له عند جلال)
4. **النتيجة عند أحمد**:
   - عند دخول أحمد لصفحة "الحسابات المتبادلة"
   - يرى جلال وبجانبه "لك عنده 500 دولار"

### سيناريو 3: العكس - أحمد يدفع لجلال

1. **جلال** يضيف حركة "واردة" لأحمد بمبلغ 300 دولار
2. **النتيجة**:
   - عند جلال: "له عليك 200 دولار" (500 - 300 = 200)
   - عند أحمد: "لك عنده 200 دولار"

## الفوائد

### 1. للمستخدمين
- إدارة واضحة للحسابات المتبادلة
- شفافية كاملة في الأرصدة
- سهولة المتابعة والمحاسبة

### 2. للمدير
- رؤية شاملة لجميع العلاقات
- إدارة كاملة للمستخدمين والعملاء
- تقارير شاملة عن نشاط النظام

### 3. للنظام
- أمان محكم مع RLS
- عدم إمكانية تداخل البيانات
- كل مستخدم يرى بياناته فقط

## اختبار النظام

### الخطوات المطلوبة للاختبار:

1. **تسجيل مستخدمين جدد**:
   - سجل مستخدمين (مثلاً: جلال، أحمد، علي)
   - لاحظ أرقام الحسابات (26003، 26004، 26005...)

2. **ربط المستخدمين**:
   - سجل دخول كـ جلال
   - أضف أحمد كعميل (ابحث برقم حسابه)
   - تأكد من ظهور أحمد في قائمة العملاء مع علامة "مستخدم"

3. **إضافة حركات**:
   - أضف حركة "صادرة" لأحمد (500 دولار)
   - تأكد من ظهور الرصيد الصحيح

4. **التحقق من الطرف الآخر**:
   - سجل دخول كـ أحمد
   - افتح صفحة "الحسابات المتبادلة"
   - تأكد من ظهور جلال مع الرصيد الصحيح

5. **اختبار المدير**:
   - سجل دخول كمدير (A)
   - تأكد من رؤية جميع العملاء والحسابات

## ملاحظات مهمة

### 1. الأمان
- جميع البيانات محمية بـ RLS
- لا يمكن للمستخدم رؤية أو تعديل بيانات مستخدم آخر
- المدير فقط لديه صلاحيات كاملة

### 2. منع التكرار
- لا يمكن ربط نفس المستخدم مرتين
- لا يمكن للمستخدم ربط نفسه كعميل

### 3. البيانات الموجودة
- جميع العملاء الحاليين تم ربطهم بحساب المدير تلقائياً
- لن تتأثر أي بيانات موجودة
